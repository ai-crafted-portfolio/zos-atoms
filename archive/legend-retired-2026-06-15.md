# 凡例 — アトムの読み方

このガイドの各ページは「**アトム**」と呼ぶ単位で書かれています。
アトムとは、ひとつの概念を「なぜ存在し・どう動き・どこで間違えるか」まで含めて
自己完結的に説明した最小の知識のかたまりです。
すべてのアトムは同じ **7 つのセクション** で構成されています。

## 7 つのセクション

| セクション | 日本語の見出し | 何が書かれているか |
| --- | --- | --- |
| purpose | なぜ存在するか | その仕組みが生まれた背景・解決している課題・設計思想 |
| mechanism | どう動くか | 内部構造・処理の流れ・主要な構成要素 |
| prerequisites | 理解の前提 | 先に読んでおくべきアトムや、必要な一般知識 |
| relations | 他アトムとの繋がり | 依存・特殊化・対比・利用関係。関連アトムへのリンク |
| pitfalls | 実装・運用での落とし穴 | 現場で頻発する失敗、典型的なエラーとその原因 |
| examples | 具体例 | JCL / IDCAMS などの実際のコード断片 |
| decision_axes | 採否を分ける判断軸 | 「A と B のどちらを選ぶか」を分ける観点 |

## relations の関係ラベル

relations セクションでは、アトム同士のつながりを次のラベルで表します。

- **depends_on** — このアトムを理解するのに前提となるアトム
- **specialized_by** — このアトムをより具体化・特殊化したアトム
- **contrasts_with** — 対比して理解すると分かりやすいアトム
- **used_by** — このアトムを利用する側のアトム

本文中に `ZOS-XXXX-001` の形で現れるアトム ID は、
対応するページがある場合はリンクになっています。クリックで移動できます。
`（未作成）` と注記された ID は、まだページ化されていない参照先です。

## 表記について

- 「アトム」「他アトムとの繋がり」「同一アトム内」といった言葉は、
  この知識整理の枠組みを指す用語です。z/OS 自体の専門用語ではありません。
- コード例は実機での動作を保証するものではなく、構文と考え方を示す教材です。

## Tier 一覧とアトム配置

新規 42 アトムは、z/OS が扱う領域を 6 つの Tier に分けて配置しています。
Tier ラベル (α / β / γ / δ / ε / ζ) は読み順の目安であり、上から順に「OS の中核」→「外側のサービス」→「運用と発展」へと広がっていきます。

| Tier | テーマ | アトム数 | 主な内容 |
| --- | --- | --- | --- |
| **α** | OS + サブシステム | 7 | ASCB / 仮想記憶 / IPL / コンソール / Wait state / MQ / WAS |
| **β** | ミドルウェア + ユーティリティ | 8 | SAF / ICSF / Pervasive Encryption / AT-TLS / 証明書管理 / IDCAMS / IEBCOPY / IEFBR14 |
| **γ** | セキュリティ + ネットワーク | 7 | RACF Advanced / ACEE / VTAM / TCP/IP / OSA / Enterprise Extender / FTP |
| **δ** | ストレージ + 監視 | 5 | DFSMShsm / Tape (VTL) / RMF / OPERLOG / NetView |
| **ε** | 復旧 + ワークロード | 6 | IPCS / RRS / ARM / IRD / Capacity planning / Batch scheduler |
| **ζ** | Sysplex + モダナイゼーション | 9 | Coupling Facility / GRS / XCF / GDPS / zCX / Linux on Z / z/OSMF / Java / Ansible |

これに既存 20 アトム (データ管理・ジョブ実行・サブシステム基礎等) を加えて、合計 62 アトムでガイドを構成しています。
左メニューの「既存アトム (20)」「新規アトム (42)」から目的の Tier・アトムへ移動できます。
