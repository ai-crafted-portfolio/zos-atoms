    ブート / NIM / mksysb¶

    bosboot （10 件）
    連番 技術項目 単一焦点要約 出典
    1437 bosboot コマンド概要 ブートイメージ (BOS ブートレコード + カーネル + 最小限のファイルシステム) を構築し、ブート論理ボリューム (hd5) に書き込むコマンド。カーネル変更・ブート LV 移動・bosboot 関連 fileset 更新の直後に必須。 AIX1, AIX73
    1438 bosboot -a 新しいブートイメージを作成して、ブート論理ボリューム (省略時 hd5) に書き込む。最も一般的な使用法。/usr/lib/boot 配下のブート関連ファイルから生成される。 AIX1, AIX73
    1439 bosboot -d device 対象デバイスを指定 (例: -d /dev/hdisk0)。複数 rootvg ディスクや代替ディスクのブートレコード更新時に必要。デバイスフルパスを指定する。 AIX1, AIX73
    1440 bosboot -l lv ブート論理ボリューム名を明示する (省略時は hd5)。multibos や alt_disk_copy の管理対象を明示する場合に使う。 AIX2, AIX73
    1441 bosboot -L 大型ブートイメージ (Large Boot Image) を許可する。RAM ディスクサイズが既定の上限を越える場合に指定する。 AIX73
    1442 bosboot -k kernel ブートイメージに組み込むカーネルを明示する (例: /usr/lib/boot/unix_64)。カーネル切替時 (32bit/64bit/MP/UP) に使用。 AIX1, AIX73
    1443 bosboot -B ブートレコードのみ書き込み、ブートイメージ自体は再生成しない。bootlist 変更後の整合性確保に使う運用パターンがある。 AIX73
    1444 bosboot -P proctype プロセッサタイプを明示してブートイメージを生成する (POWER アーキ別の最適化用)。通常は不要だが、移行用イメージ生成で利用。 AIX73
    1445 bosboot -s ブートイメージのサイズ計算 / 検証のみ行い、実際の書き込みは行わない。事前確認用。 AIX73
    1446 bosboot 実行タイミング カーネル変更 (32→64bit 等)、ブート LV の物理位置移動、デバイスドライバ ODM 変更、bos.rte.boot/bos.rte.install 更新の直後は必ず実行が必要。 AIX1, AIX2, AIX73
    bootlist （10 件）
    連番 技術項目 単一焦点要約 出典
    1447 bootlist コマンド概要 システム起動時に NVRAM が参照するブートデバイス順序を表示・変更する。normal / service / both / prevboot モードを操作可能。 AIX1, AIX73
    1448 bootlist -m normal 通常 (multi-user) 起動用のブートデバイスリストを操作する。例: bootlist -m normal hdisk0 hdisk1。 AIX1, AIX73
    1449 bootlist -m service サービス (診断/メンテナンス) モード起動用のブートデバイスリストを操作する。NIM / CD / tape からの起動を仕込む場合に使う。 AIX1, AIX2, AIX73
    1450 bootlist -m both normal と service の両方を同一リストで設定する。シンプルな構成で運用を統一したい場合に使う。 AIX73
    1451 bootlist -m prevboot 直前のブートデバイスを表示する。マルチパス/代替ディスク運用での切り戻し検証に利用。 AIX73
    1452 bootlist -o 現在設定されているブートデバイスリストを表示する。読み取り専用。 AIX1, AIX73
    1453 bootlist -v 詳細表示。論理デバイス名に加え、blv (ブート LV) や属性も併せて表示する。 AIX73
    1454 bootlist デバイスリスト引数 hdisk0 hdisk1 ... をスペース区切りで並べた順がブート優先順位になる。最大数はハード/モデル依存 (典型 5 個程度)。 AIX1, AIX73
    1455 bootlist サービスモード用途 diag / NIM ブート / CD ブート 等を service モードに仕込むと、F5/F6 等で起動した際にサービス側リストが使われる。 AIX1, AIX2, AIX73
    1456 bootlist NIM 経由ブート サービスリストに NIM Master を解決できるアダプタを設定すれば、ネットワークブートで bos_inst / diag / maintenance を起動可能。 AIX2, AIX73

    inittab （16 件）
    連番 技術項目 単一焦点要約 出典
    1463 /etc/inittab 構文 形式は Identifier:RunLevel:Action:Command の 4 カラム。Identifier は最大 14 文字。AIX 専用 action (boot/bootwait/sysinit) が存在する点が特徴。 AIX1, AIX73
    1464 inittab action: respawn プロセスが終了したら init が自動的に再起動する。getty / ttymon 系で多用。 AIX1, AIX73
    1465 inittab action: wait 指定ランレベルに入った時に 1 回だけ実行し、終了を待つ。サブシステム初期化で典型。 AIX1, AIX73
    1466 inittab action: once 指定ランレベルで 1 回だけ実行し、終了は待たない。バックグラウンドのデーモン投入に使う。 AIX1, AIX73
    1467 inittab action: boot システムブート時に 1 回実行 (ランレベルに依存しない)。終了を待たない。AIX 特有。 AIX1, AIX73
    1468 inittab action: bootwait システムブート時に 1 回実行し、終了を待ってから次へ進む。AIX 特有。マウントなど順序依存の初期化に使う。 AIX1, AIX73
    1469 inittab action: sysinit init が他の処理を始める前 (コンソールアクセス前) に実行される。最初期初期化用。 AIX1, AIX73
    1470 inittab action: initdefault init 起動時に遷移する既定ランレベルを定義する。Command 欄は使わず、ランレベル欄の値が使用される。 AIX1, AIX73
    1471 inittab action: powerfail SIGPWR (電源異常) 受信時に実行。UPS 連携や緊急シャットダウンスクリプトを差し込む。 AIX2, AIX73
    1472 inittab action: powerwait powerfail と同様だが完了を待つ。電源異常時に同期的な後処理が必要な場合に使用。 AIX73
    1473 inittab action: off エントリを無効化する。削除せず一時的に止める場合に使う。 AIX1, AIX73
    1474 mkitab /etc/inittab に新規行を追加する公式コマンド。直接 vi 編集よりも mkitab / chitab / rmitab を推奨。 AIX1, AIX73
    1475 chitab /etc/inittab の既存エントリを変更する。Identifier をキーに動作 / コマンドを書き換える。 AIX1, AIX73
    1476 lsitab /etc/inittab の内容を表示する。lsitab -a で全件表示、lsitab id で 1 件表示。 AIX1, AIX73
    1477 rmitab /etc/inittab から指定 Identifier の行を削除する。 AIX1, AIX73
    1478 inittab 編集の注意 壊れた inittab は init が起動を中断する原因になる。必ず lsitab / mkitab 系コマンドを使い、編集前にバックアップを取る。 AIX1, AIX73
