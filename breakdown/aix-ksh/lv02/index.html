    JFS / JFS2 ファイルシステム¶

    製品: AIX + ksh ／ 本ページ件数: 180 件 ／ 出典区分: 書籍出典

    FS作成 - crfs （21 件）
    連番 技術項目 単一焦点要約 出典
    233 crfs 概要 crfs は /etc/filesystems にスタンザを登録した上で論理ボリュームを作成し JFS/JFS2 ファイルシステムを生成する高水準コマンド。mkfs と異なり LV 未指定でも自動的に LV を確保する。 AIX73, AIX1
    234 crfs -v vfstype vfs タイプを指定する必須オプション。jfs2 (推奨), jfs, nfs, cdrfs などを指定する。jfs2 は 64bit inode で 16TB 超に対応。 AIX73, AIX1
    235 crfs -g VolumeGroup ファイルシステムを作成する VG を指定する。-d で既存 LV を指定する場合は -g は不要。-g 指定時は LV が自動作成される。 AIX73, AIX1
    236 crfs -d Device 既存の論理ボリューム名を指定する。-g と排他。LV は事前に mklv で作成しておく必要があり、crfs はそこに FS 構造のみ書き込む。 AIX73, AIX1
    237 crfs -m MountPoint マウントポイントを絶対パスで指定する。指定したディレクトリが存在しなければ crfs が自動作成する。/etc/filesystems の mount スタンザのキーになる。 AIX73, AIX1
    238 crfs -A yes|no システム起動時 (mount all) に自動マウントするかを指定する。yes で /etc/filesystems に mount = true が書かれる。 AIX73, AIX1
    239 crfs -p rw|ro デフォルトの読み書き許可。rw は読み書き、ro は読み取り専用。/etc/filesystems の type フィールドではなく account/options に反映。 AIX73
    240 crfs -a size=Value ファイルシステムサイズを指定。単位は 512B ブロック、または末尾に M (MB), G (GB), T (TB) を付ける。先頭に + を付けると相対増分指定。例: -a size=2G。 AIX73, AIX1
    241 crfs -a bf=true (JFS only) JFS で 2GB 超の Large File Enabled ファイルシステムを作成する。JFS2 では常に大ファイル対応のため不要。 AIX73, AIX1
    242 crfs -a logname=LV JFS/JFS2 ログ用 LV を明示指定する。指定しない場合 VG 内の既定 jfs2log が使用される。INLINE log (JFS2 専用) を使う場合は logname=INLINE を指定する。 AIX73
    243 crfs -a log=INLINE (JFS2) JFS2 のインラインログ機能。ファイルシステム本体と同一 LV 内にログ領域を埋め込み、別ログ LV を不要にする。-a logsize で領域確保。 AIX73
    244 crfs -a logsize=Value (JFS2) INLINE ログのサイズを MB 単位で指定。最小 4MB、推奨 0.4% 程度。FS 拡張時のログ枯渇対策に十分な値を確保する。 AIX73
    245 crfs -a ea=v1|v2 (JFS2) 拡張属性 (Extended Attributes) のフォーマットバージョン。v2 は ACL/AIXC/NFS4 ACL と任意名前空間の EA をサポート。v1 は古い形式で機能制限あり。 AIX73
    246 crfs -a quota=userquota,groupquota クォータを有効化するスタンザを追加する。userquota / groupquota / userquota,groupquota を指定可能。quotaon コマンドで実際にクォータ強制を開始する。 AIX73
    247 crfs -a frag/nbpi/ag (JFS only) JFS 固有チューニング。frag=512/1024/2048/4096 (フラグメントサイズ)、nbpi=N (Number of Bytes Per Inode、小値で inode 多数)、ag=8/16/32/64 (Allocation Group MB)。JFS2 では bsize=4096 固定・inode 動的のため不要。 AIX73, AIX1
    248 crfs -a compress=lzs (JFS2) JFS2 のオンライン圧縮を有効化。LZS アルゴリズムで透過圧縮。読み取り時に展開、書き込み時に圧縮。CPU 負荷増・I/O 性能低下に注意。 AIX73, AIX2
    249 crfs -a efs=yes (JFS2) EFS (Encrypted File System) を有効化する。FS 単位で暗号化メタデータを管理し、ユーザ鍵で各ファイル暗号化を行う。 AIX73
    250 crfs -a isnapshot=yes (JFS2) 内部スナップショット (internal snapshot) 機能を有効化したファイルシステムとして作成する。専用スナップショット LV を不要にする。 AIX73
    251 crfs -a vix=yes (JFS2) 可変 inode 拡張 (variable inode extension) を許可。inode 領域が動的に拡張される。 AIX73
    252 crfs -a mountguard=yes (JFS2) PowerHA / 共有 VG 環境で同一 FS の二重マウントを阻止する Mount Guard を有効化。マウント時に他ノード使用を検出すれば失敗させる。 AIX73
    253 crfs -u MountGroup マウントグループ名を /etc/filesystems の mount スタンザに付与し、mount -t Group で同一グループの FS を一括マウント可能にする。 AIX73
    FS作成 - mkfs （10 件）
    連番 技術項目 単一焦点要約 出典
    254 mkfs 概要 mkfs は低水準 FS 作成コマンド。LV を事前に mklv で作成しておく必要があり、/etc/filesystems への登録は行わない (crfs と対照的)。 AIX73, AIX1
    255 mkfs -V vfstype ファイルシステムタイプを指定。jfs (旧 JFS), jfs2 (推奨)。/etc/vfs に定義された vfs エントリ名と一致させる。 AIX73
    256 mkfs -L Label ボリュームラベルを設定する。lsfs や fsck で参照可能。 AIX73
    257 mkfs -m MountPoint マウントポイントをメタデータに記録する。/etc/filesystems への登録は行わない。 AIX73
    258 mkfs -s Size FS サイズを 512B ブロック単位で指定する。LV の物理サイズより小さく設定可能。 AIX73
    259 mkfs -o Options vfs 固有のオプションを渡す。例: -o nbpi=4096, -o log=/dev/loglv00。 AIX73
    260 mkfs.jfs / mkfs.jfs2 vfs 種別ごとのフロントエンド。mkfs -V jfs2 と同等。直接呼ぶ場合は mkfs.jfs2 -O log=INLINE のようにオプション付与可。 AIX73
    261 mkfs.jfs2 -O log_partition_count JFS2 ログ用パーティション数。INLINE ログ使用時に内部割り当て数を制御する。 AIX73
    262 mkfs.jfs2 -O outline / inline log ログ配置を outline (外部ログ LV) / inline (本体 LV 内) のいずれかで指定。inline はバックアップ・移行が楽。 AIX73
    263 mkfs.jfs2 -O isnapshot=yes FS 作成時に内部スナップショットを有効化する。 AIX73
    /etc/filesystems （12 件）
    連番 技術項目 単一焦点要約 出典
    264 スタンザ構造 /etc/filesystems はマウントポイントをスタンザ名としたフラットテキスト。各スタンザは属性 = 値 で記述し、空行で区切る。 AIX73, AIX1
    265 dev 属性 dev = /dev/lvname の形式で論理ボリュームデバイスを指定する。FS 実体への入口。 AIX73, AIX1
    266 vfs 属性 vfs = jfs2 / jfs / nfs / cdrfs / procfs / autofs などのファイルシステム種別。/etc/vfs に対応エントリが必要。 AIX73, AIX1
    267 log 属性 log = /dev/jfs2log (例) で外部ログ LV を指定する。INLINE 時は log = INLINE。 AIX73
    268 mount 属性 mount = true|false|automatic|removable。true で mount -a 対象、automatic でブート時自動マウント、false で手動のみ。 AIX73, AIX1
    269 check 属性 check = true|false|順序番号。fsck -p 時の検査対象と並列度を制御する。 AIX73
    270 type 属性 type = MountGroup。mount -t Group で一括マウントするためのグループ名。 AIX73
    271 options 属性 options = rw,noatime,nodev など mount -o 相当のオプションをカンマ区切りで指定する。 AIX73
    272 account 属性 account = true|false。ディスク利用統計 (diskusg) の対象とするか。 AIX73
    273 quota 属性 quota = userquota,groupquota。クォータ強制対象種別。 AIX73
    274 nodename 属性 (NFS) nodename = server。NFS マウント時にサーバ FQDN を指定する。 AIX73
    275 free 属性 (JFS) free = true|false。FS のフォーマット直後を示すフラグ。 AIX73
    FS変更 - chfs （14 件）
    連番 技術項目 単一焦点要約 出典
    276 chfs 概要 chfs はマウント済み or 未マウントの JFS/JFS2 ファイルシステム属性を変更する。LV サイズ拡張・mount フラグ変更・ログ変更などに用いる。 AIX73, AIX1
    277 chfs -A yes|no 自動マウント属性を変更する。/etc/filesystems の mount 値を書き換える。 AIX73
    278 chfs -a size=Value ファイルシステム拡張。+ で相対増分指定可 (例: -a size=+1G)。JFS2 ではオンライン拡張が可能。 AIX73, AIX1
    279 chfs -a size=- (オンライン縮小) JFS2 のみ対応するオンライン縮小。-a size=-1G で 1GB 縮小、または絶対値で目標サイズを指定。inode/ブロック移動が発生するため時間を要する。 AIX73
    280 chfs -m NewMountPoint マウントポイントを変更する。マウント中は変更不可、umount 後に実行する。 AIX73
    281 chfs -d Attribute /etc/filesystems から指定属性を削除する。例: chfs -d account /home。 AIX73
    282 chfs -u MountGroup マウントグループ (type 属性) を変更する。 AIX73
    283 chfs -a logname=LV FS が使用するログ LV を変更する。アンマウント状態で実行する必要がある。 AIX73
    284 chfs -a quota=... クォータ種別の有効/無効化。userquota / groupquota / 両方を指定。 AIX73
    285 chfs -a ea=v2 (JFS2) 拡張属性フォーマットを v1 から v2 へ変換する。一旦 v2 にすると v1 に戻せない。 AIX73
    286 chfs -a splitcopy=/MP -a copy=N (スプリットミラー) ミラー化された JFS2 FS のオフライン複製を作成する。指定したミラーコピーを切り出して読み取り専用 FS としてマウント可能にする。 AIX73
    287 chfs -a freeze=N / refreeze=N / thaw FS を瞬時凍結する。バックアップ整合性確保用。freeze=秒数で自動解凍、thaw で即時解除。 AIX73
    288 chfs -a options=... mount オプション (rw, noatime, log=NULL など) を変更する。 AIX73
    289 chfs -a mountguard=yes/no Mount Guard を後付け有効化・無効化する。 AIX73
