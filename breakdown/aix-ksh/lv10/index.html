    診断 (errpt/snap) + PowerHA¶

    製品: AIX + ksh ／ 本ページ件数: 216 件 ／ 出典区分: 書籍出典

    errpt （26 件）
    連番 技術項目 単一焦点要約 出典
    1893 errpt AIX のエラーログ表示コマンド。errdemon が /var/adm/ras/errlog に書いた二進ログを整形して標準出力に出す。オペランド無しでサマリ一覧 (識別子/タイムスタンプ/タイプ/クラス/リソース/説明) を表示。 AIX2, AIX73
    1894 errpt -a Detailed format。各エントリのテンプレートに従い詳細表示。原因/推奨処置/障害データ (Detail Data) を全部出す。サポート問い合わせの基本情報。 AIX2, AIX73
    1895 errpt -d エラークラスでフィルタ。H=Hardware (PdAt 上のハードウェア)、S=Software、O=Operator (errlogger 由来)、U=Undetermined を組合せ可。例 -d H,S。 AIX2, AIX73
    1896 errpt -j 指定 ID (8桁16進、例 AA8AB241) のエントリだけ表示。-j A,B,C のようにカンマ列挙可。errpt -a -j がトラブルシュート定番。 AIX2, AIX73
    1897 errpt -k -j の逆。指定 ID を除外して表示。よく出るがどうでもいい情報 (例 SRC リスタート) を黙らせるのに使う。 AIX2, AIX73
    1898 errpt -t テンプレートを表示。エラーログ ID 毎に定義された Label / Class / Type / Description を確認する。errdemon が参照する /etc/objrepos/errtmplt と対応。 AIX2, AIX73
    1899 errpt -l シーケンス番号 (errdemon が付与する連番) でエントリ指定。-a と組み合わせて 1 件詳細表示。 AIX2, AIX73
    1900 errpt -N リソース名 (例 hdisk0 / ent0 / sysplanar0) でフィルタ。-N hdisk0 で当該ディスクのエラーだけ。 AIX2, AIX73
    1901 errpt -n リソースクラス (例 disk, adapter, memory, processor) でフィルタ。-N より広く絞る。 AIX2, AIX73
    1902 errpt -s 開始日時を mmddhhmmyy (月日時分年) 形式で指定。例 -s 0601000026 → 2026/06/01 00:00 以降。 AIX2, AIX73
    1903 errpt -e 終了日時を mmddhhmmyy 形式で指定。-s と -e を併用して期間絞り込み。 AIX2, AIX73
    1904 errpt -T エラー種別でフィルタ。PEND (障害発生予兆)、PERF (性能低下)、PERM (回復不能)、TEMP (一時)、UNKN、INFO の組合せ。例 -T PERM,PEND。 AIX2, AIX73
    1905 errpt -A Concurrent (abbreviated) 形式。1 行サマリの圧縮版。シェル集計で扱いやすい。 AIX2, AIX73
    1906 errpt -V Very detailed。-a よりさらに詳しくテンプレ情報と詳細データを並べる。 AIX2, AIX73
    1907 errpt -y 別 errlog バイナリファイル (例 別ホストからコピーした /var/adm/ras/errlog) を読む。スナップショット解析の必須オプション。 AIX2, AIX73
    1908 errlogger "" オペレータ起源メッセージを errlog に書く。シェルスクリプトから故障マークを残す用途。OPMSG ラベル/O クラスで記録される。 AIX2, AIX73
    1909 errclear エラーログから N 日より古いエントリを削除。例 errclear 30 → 30 日より古いものを削除。日次 cron に入れて肥大化抑制。 AIX2, AIX73
    1910 errclear -d/-j オプション クラス指定削除 (errclear -d H 0 でハードウェア全削除) や ID 指定削除 (-j ) で煩い特定種別だけ消す。 AIX2, AIX73
    1911 errstop errdemon を停止。停止中は errlog 書き込み不可。通常は errdemon を常駐させる。errlog ファイルメンテ時のみ停止。 AIX2, AIX73
    1912 errdemon エラーログデーモン。/usr/lib/errdemon。boot 時に rc.boot から起動。kernel が posixyieldlog 経由で渡したイベントを /var/adm/ras/errlog に書く。 AIX2, AIX73
    1913 errdemon -B errlog ファイル最大サイズ設定 (バイト)。既定 1MB。/var/adm/ras/errlog サイズが上限超で巻き戻す。 AIX2, AIX73
    1914 errdemon -l 現在の errdemon 設定 (errlog パス/サイズ/buffer) 表示。 AIX2, AIX73
    1915 errupdate エラーテンプレ ODM (errtmplt) を更新。.err ファイル (テンプレ定義) を入力に取り、エラー ID 追加/削除/更新。アプリ側がカスタムエラーを定義する時に使う。 AIX2, AIX73
    1916 errnotify ODM クラス エラー発生時の自動アクション (メール送信/スクリプト起動) を定義する ODM クラス。/etc/objrepos/errnotify。en_name / en_persistenceflg / en_method 等を持つ。 AIX2, AIX73
    1917 /var/adm/ras/errlog エラーログ本体 (バイナリ)。errpt はこれを読む。直接編集禁止。サイズ上限は errdemon -B で制御。 AIX2, AIX73
    1918 /var/adm/ras/conslog コンソール出力のテキストログ。boot 中や hard error 時に console に出たメッセージを保持。 AIX2, AIX73
    syslog （10 件）
    連番 技術項目 単一焦点要約 出典
    1919 syslog BSD 由来のシステムログサービス。kernel/デーモン/アプリが syslog() API でログを送出し syslogd が振り分け。AIX では errpt とは別系統 (errpt はバイナリ ODM テンプレ式、syslog はテキスト式)。 AIX2, AIX73
    1920 syslogd /usr/sbin/syslogd。SRC で管理される subsystem。startsrc -s syslogd / stopsrc -s syslogd / refresh -s syslogd で操作。/etc/syslog.conf を読む。 AIX2, AIX73
    1921 /etc/syslog.conf syslogd 設定ファイル。1 行 1 ルールで .[;...] 形式。タブ区切り必須 (スペースだと動かない事例あり)。 AIX2, AIX73
    1922 syslog facility 送信元区分。kern / user / mail / daemon / auth / syslog / lpr / news / uucp / cron / local0..local7 / * (全て)。 AIX2, AIX73
    1923 syslog priority 重要度。emerg / alert / crit / err / warning / notice / info / debug / none。指定 priority 以上を捕捉。 AIX2, AIX73
    1924 syslog action 出力先。/path/to/file (絶対パス)、@hostname (UDP 転送)、user1,user2 (端末通知)、* (全ログインユーザ)、|prog (パイプ)。 AIX2, AIX73
    1925 syslog.conf 記述例 .err /var/adm/messages で全 facility の err 以上をファイル保存。.* @loghost で別ホスト syslogd (UDP 514) へ転送 (集中ログホスト構成)。 AIX2, AIX73
    1926 refresh -s syslogd syslog.conf 変更後に syslogd へ SIGHUP を送り設定再読込。停止再開せずに済む標準手順。 AIX2, AIX73
    1927 logger コマンド シェルから syslog にメッセージ送出。logger -p user.info "message" 等で facility.priority 指定。シェルスクリプトの監査痕跡用。 AIX2, AIX73
    1928 rotate オプション syslog.conf 行末に rotate size M files time d で AIX 固有のローテーション設定可能。logrotate なしで完結。 AIX73
    alog （10 件）
    連番 技術項目 単一焦点要約 出典
    1929 alog AIX 固有の循環ログ機構。boot/console/nim/cfg/dumpsymp 等の種別毎に固定サイズの循環バッファをファイルとして持つ。サイズ超過で先頭から上書き。 AIX2, AIX73
    1930 alog -L 登録済みログ種別 (boot / console / nim / cfg 等) と対応ファイル / サイズ / 説明を一覧表示。どの alog 種別が有効か確認する。 AIX2, AIX73
    1931 alog -t ログ種別指定。-t boot で起動ログ、-t console でコンソール、-t nim で NIM、-t cfg で cfgmgr 系。 AIX2, AIX73
    1932 alog -o ログを表示。alog -t boot -o で boot ログ全文出力。トラブル時の基本コマンド。 AIX2, AIX73
    1933 alog -p コマンド出力を alog にパイプ書き込み。例 echo "event" | alog -t boot -p cat。プログラムから alog に書く側。 AIX2, AIX73
    1934 alog -s/-C -s でサイズ変更 (バイト、既定 boot は約 16KB)、-C で属性を ODM 登録 (永続化)。組合せで再起動後も維持。 AIX73
    1935 alog -t boot 確認 起動時のメッセージ全文。LED コード遷移、cfgmgr 出力、rc.boot 各 phase の処理が記録。boot 失敗解析の出発点。 AIX2, AIX73
    1936 alog -t console コンソールに表示されたシステムメッセージ全文。errpt と二系統で取得することで取りこぼし防止。 AIX2, AIX73
    1937 alog -t nim NIM (Network Installation Manager) クライアント側のインストール/復元動作ログ。 AIX2, AIX73
    1938 alog -t cfg cfgmgr 起動時のデバイス検出ログ。新規 LUN マッピング後の cfgmgr -v 結果も残る。 AIX73

    sysdump （15 件）
    連番 技術項目 単一焦点要約 出典
    1941 sysdumpdev システムダンプデバイス設定/状態確認コマンド。primary / secondary / autoStart / always Allow Dump / dump copy directory などを管理。 AIX2, AIX73
    1942 sysdumpdev -l 現在の dump 設定を表示。primary device, secondary device, copy directory, forced copy flag, always allow dump 等が一覧で出る。 AIX2, AIX73
    1943 sysdumpdev -p Primary dump device を設定。例 sysdumpdev -p /dev/lg_dumplv。次回再起動までの一時設定 (永続化には -P 併用)。 AIX2, AIX73
    1944 sysdumpdev -s Secondary dump device を設定。primary が容量不足/書込不能時のフォールバック。 AIX2, AIX73
    1945 sysdumpdev -P 永続化フラグ。-p / -s と併用して /etc/swapspaces や ODM (CuAt) に書き込み、再起動後も維持。 AIX2, AIX73
    1946 sysdumpdev -e 次回 dump に必要な見積もりサイズをバイト表示。dump device サイズと比較して不足なら拡張要。 AIX2, AIX73
    1947 sysdumpdev -d boot 時に dump を自動コピーする先ディレクトリを設定。コピー失敗時はオリジナル dump を破棄。 AIX2, AIX73
    1948 sysdumpdev -D -d と同じ自動コピー指定。ただしコピー失敗時はオリジナル dump を保持。-D を推奨。 AIX2, AIX73
    1949 sysdumpdev -L 最新の dump 情報 (発生日時 / dump device / 推定サイズ / 終了コード) を表示。dump 取得後の最初の確認。 AIX2, AIX73
    1950 sysdumpdev -K Always Allow Dump (キー操作で dump 強制取得) を有効化。HMC の Dump キー押下に応答するか。 AIX2, AIX73
    1951 sysdumpdev -Z 圧縮 dump (Compressed) サイズと dump device 容量を比較表示。 AIX73
    1952 sysdumpstart コマンドからシステムダンプを強制取得。-p / -s で primary/secondary 指定。実行直後にシステムは halt → ipl。本番では計画停止時のみ。 AIX2, AIX73
    1953 sysdumpstart -p Primary dump device に dump を書いて再起動。 AIX2, AIX73
    1954 sysdumpstart -s Secondary dump device に dump を書いて再起動。 AIX2, AIX73
    1955 /dev/lg_dumplv AIX が既定で作る dump 用 LV (rootvg 上の Logical Volume)。種別 sysdump。サイズはメモリと相関するので増設後は要拡張。 AIX73
    snap （11 件）
    連番 技術項目 単一焦点要約 出典
    1956 snap サポート診断バンドル作成コマンド。errpt/lsattr/lsdev/lvm/network 等を一括収集し /tmp/ibmsupt 配下に保存。IBM サポート送付の標準入力物。 AIX2, AIX73
    1957 snap -a All。全コンポーネント (general/kernel/lvm/nfs/printer/sna/ssa/tcpip/...) を収集。情報が多くて時間と空き容量を要する。 AIX2, AIX73
    1958 snap -c 収集後にバンドルを圧縮 (pax + compress) して snap.pax.Z を作成。サポート送付前提なら通常 -c を付ける。 AIX2, AIX73
    1959 snap -g General 情報のみ収集。/etc/inittab、lslpp -L、lsps -a、df、lsattr、bootlist、errpt -a など。 AIX2, AIX73
    1960 snap -k / -L / -n / -t / -i 個別コンポーネント収集: -k Kernel (パラメタ/locked memory)、-L LVM (lspv/lsvg/lslv)、-n NFS (nfsstat/exports)、-t TCP/IP (netstat/ifconfig/no -a)、-i Installation。組み合わせ可。 AIX2, AIX73
    1961 snap -d 出力ディレクトリ指定。既定 /tmp/ibmsupt。空きが少ない時は別 FS を指定。 AIX2, AIX73
    1962 snap -o / -r -o でテープ等デバイスへ直接書き出し (例 snap -o /dev/rmt0)。-r で /tmp/ibmsupt 配下の前回 snap 出力を削除。 AIX2, AIX73
    1963 snap -gc general + 圧縮 を 1 発で。日常トラブル送付の最短コマンド。 AIX2, AIX73
    1964 snap -e PowerHA SystemMirror (旧 HACMP) バンドル収集。/var/hacmp/log 配下や clstrmgr 構成情報をまとめる。クラスタ障害解析の基本。 PHA7, AIX73
    1965 /tmp/ibmsupt snap の出力先ディレクトリ。各コンポーネント (general/lvm/tcpip/nfs/...) 別サブディレクトリと、-c 指定時の snap.pax.Z。 AIX2, AIX73
    1966 snap.pax.Z snap -c で生成される圧縮アーカイブ。IBM サポートにアップロードする実体。 AIX2, AIX73
    kdb （8 件）
    連番 技術項目 単一焦点要約 出典
    1967 kdb AIX カーネルデバッガ。生システムでは BOS インストール時に Allow Online Diagnostics で有効化。dump 解析モードでも使う。 AIX2, AIX73
