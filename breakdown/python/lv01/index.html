    識別子と予約語 （5 件）
    連番 技術項目 単一焦点要約 出典
    1 識別子の命名規則 識別子は ASCII 英字・数字・アンダースコアおよび Unicode の Letter カテゴリで構成される。先頭は数字不可。PEP 3131 により非 ASCII 識別子も合法だが、慣習として ASCII を推奨。 PYDOC
    2 予約語 (keyword) 一覧 False/None/True/and/as/assert/async/await/break/class/continue/def/del/elif/else/except/finally/for/from/global/if/import/in/is/lambda/nonlocal/not/or/pass/raise/return/try/while/with/yield/match/case (soft)。keyword.kwlist で取得可能。 PYDOC
    3 soft keyword (match/case/type) match/case は構文上のキーワードだが、文脈外では識別子としても使用可能 (soft keyword)。type も 3.12 から PEP 695 構文の soft keyword 扱い。 PYDOC
    4 PEP 8 命名規約 クラスは CapWords、関数/変数は snake_case、定数は UPPER_SNAKE、モジュールは小文字、内部利用は先頭 _、name mangling は __name (末尾無し)。 PYDOC
    5 dunder 名の意味 前後 2 個のアンダースコアで囲まれた名前は Python が予約する特殊名。init, str, getitem などのプロトコルメソッドや name, file などのモジュール属性。 PYDOC

    行とインデント （5 件）
    連番 技術項目 単一焦点要約 出典
    9 論理行と物理行 Python は改行で論理行が区切られる。バックスラッシュで行継続するか、() [] {} 内では暗黙的に複数物理行に分割できる。 PYDOC
    10 インデントによるブロック ブロックはコロン + 改行 + 一段深いインデントで表現。空白とタブの混在は禁止 (TabError)。PEP 8 はスペース 4 個を推奨。 PYDOC
    11 セミコロンによる複数文 1 行に複数文を書くときは ; で区切る。PEP 8 では非推奨だがワンライナーで使用されることがある。 PYDOC
    12 暗黙の行継続 ( ) [ ] { } 丸括弧/角括弧/波括弧の内側では改行しても構文エラーにならない。長い式や引数リストの折り返しに使う。 PYDOC
    13 明示的行継続 \ 行末のバックスラッシュは次行と論理的に同一行と扱う。括弧で囲めるなら不要。 PYDOC

    14 整数リテラル 10/0x/0o/0b 通常 10 進、0x で 16 進、0o で 8 進、0b で 2 進。アンダースコアで桁区切り可能 (1_000_000)。 PYDOC
    15 浮動小数点 1.5 / 1e3 IEEE 754 倍精度。指数表記 1e3、1.5e-2 が可能。float('inf'), float('nan') で無限大/NaN を生成。 PYDOC
    16 複素数 1+2j 末尾 j で虚数部。complex(1, 2) でも生成可能。.real, .imag, abs(), conjugate() を持つ。 PYDOC
    17 数値リテラルのアンダースコア区切り PEP 515 で整数/小数とも 1_000_000 のように _ を挟める。可読性向上目的で値は変わらない。 PYDOC
    18 True / False の整数互換 bool は int のサブクラスで True == 1, False == 0。算術演算で int として扱える。 PYDOC

    39 算術演算子 + - * / // % ** / は真の除算 (常に float)、// は商の床、% は剰余、** はべき乗。3 ** 0.5 のように非整数指数も可。 PYDOC
    40 divmod(a, b) 商と剰余をタプルで返す。(a // b, a % b) と等価だが 1 回の演算で済む。 PYDOC
    41 比較演算子 == != < <= > >= is is not is は同一オブジェクトかの参照比較、== は eq による値比較。None との比較は is None が慣習。 PYDOC
    42 比較の連鎖 0 < x < 10 Python では a < b < c が a < b and b < c と等価に解釈される。 PYDOC
    43 論理演算子 and / or / not 短絡評価。and/or は最後に評価されたオペランドを返すため bool 以外でも値として使える (デフォルト値の慣用句)。 PYDOC
    44 ビット演算子 & | ^ ~ << >> 整数のビット単位 AND/OR/XOR/NOT/左シフト/右シフト。int は任意精度なので左シフトでも桁あふれしない。 PYDOC
    45 セイウチ演算子 := PEP 572。式の中で代入を行う。while (一次資料 := f.read(4096)): が典型用法。 PYDOC
    46 条件式 a if cond else b 三項演算子相当。値を返す式として if 文と区別される。 PYDOC
    47 in / not in コンテナのメンバシップ。contains が定義されていれば呼び出される。dict は key を検索。 PYDOC
    48 拡張代入 += -= = /= //= %= *= &= |= ^= >>= <<= iadd などが定義されていればインプレース更新、なければ x = x + y と同じ動作。 PYDOC
    49 演算子優先順位 高→低: ** > 単項+-~ > * / // % @ > + - > シフト > & > ^ > | > 比較/is/in > not > and > or > 条件式 > lambda。括弧で明示を推奨。 PYDOC

    str メソッド （19 件）
    連番 技術項目 単一焦点要約 出典
    51 str.upper() / lower() / swapcase() / casefold() 大小変換。casefold はドイツ語 ß などを比較用にさらに正規化する点が lower との違い。 PYDOC

    54 str.removeprefix() / removesuffix() PEP 616 (3.9+)。先頭/末尾と一致したら除去、不一致ならそのまま返す。strip 系の罠 (文字集合扱い) を避ける目的。 PYDOC
    55 str.split() / rsplit() 区切りで分割。maxsplit 指定可能。区切り省略時は連続する空白を 1 区切り扱い。 PYDOC
    56 str.splitlines() 改行コード (\r\n, \n, \r, \v 等) を Unicode 仕様に従って認識して分割。keepends=True で改行を残す。 PYDOC
    57 str.join(iterable) iterable の文字列を区切り文字で結合。','.join(map(str, lst)) が定石。 PYDOC
    58 str.replace(old, new, count) 部分置換。count を指定すると先頭から count 回まで置換。re.sub と比べて固定文字列で高速。 PYDOC
    59 str.find() / rfind() / index() / rindex() 部分文字列の位置を返す。find は見つからないと -1、index は ValueError。 PYDOC
    60 str.count(sub) 重ならない出現回数を返す。 PYDOC
    61 str.startswith() / endswith() プレフィックス/サフィックス判定。タプルで複数候補を一度に判定可能。 PYDOC
    62 str.isascii() / isalpha() / isdigit() / isalnum() / isspace() / isnumeric() / isdecimal() 文字種判定。空文字列は False。isdigit/isnumeric/isdecimal は Unicode 数値の扱いが微妙に異なる。 PYDOC
    63 str.encode(encoding, errors) str → bytes。errors='strict'|'ignore'|'replace'|'xmlcharrefreplace'|'backslashreplace'。 PYDOC
    64 str.format() / str.format_map() {} を引数で置換。format_map は辞書を unpack せず渡すため大規模辞書で高速。 PYDOC
    65 str.translate(table) / str.maketrans() 文字単位置換テーブル。maketrans で table を構築。1 文字 → 1 文字 / 削除をまとめて高速処理。 PYDOC
    66 str.zfill(width) / ljust / rjust / center ゼロパディングや左右中央寄せ。format spec で書ける場合も多い。 PYDOC
    67 str.partition() / rpartition() 最初(最後)の区切りで 3 分割し (head, sep, tail) のタプルを返す。区切りが無ければ (s, '', '')。 PYDOC
    68 str.expandtabs(tabsize) タブを空白に展開。デフォルト 8。 PYDOC
    69 str スライス s[a:b:step] 新しい文字列を返す。負ステップで反転 (s[::-1])。文字列はイミュータブルなので逐次連結は join() を使う。 PYDOC
