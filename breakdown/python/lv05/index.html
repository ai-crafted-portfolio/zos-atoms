    526 try / except 基本 try ブロックで発生した例外を except 節で捕捉する。複数 except を並べて型別に処理可。 PYDOC
    527 except 例外型 as e 捕捉した例外インスタンスを名前 e に束縛する。except 節終了時に名前は削除される (内部のスコープリーク防止)。 PYDOC
    528 try / except / else else 節は try が例外なく完了したときだけ実行される。本処理と例外処理を明確に分離。 PYDOC
    529 try / except / finally finally は例外の有無に関わらず必ず実行。リソース解放に使うが with の方が簡潔。 PYDOC
    530 try / except / else / finally の混合 順序は try → except → else → finally。すべてを同時に書くことも可能。 PYDOC
    531 bare except: の問題点 全例外を握り潰し KeyboardInterrupt / SystemExit まで拾うため Ctrl-C や exit() が効かなくなる。 PYDOC
    532 except Exception BaseException を除いた通常例外を全て拾う安全な書き方。ロギング後 raise が定石。 PYDOC
    533 複数型を 1 節で捕捉 except (KeyError, IndexError) タプルで複数の型を 1 つの節にまとめる。 PYDOC
    534 サブクラスとの一致 except 節の型はサブクラスにも一致する。FileNotFoundError は except OSError でも捕捉される。 PYDOC
    535 例外伝播の流れ 捕捉されない例外は呼び出しスタックを巻き戻りつつ伝播し、最終的にインタプリタが unhandled exception を出力。 PYDOC
