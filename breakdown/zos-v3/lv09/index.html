    TSO / ISPF / SDSF¶

    製品: z/OS v3 ／ 本ページ件数: 355 件 ／ 出典区分: Redbook ingest 反映済 v3

    TSO_ALLOCATE （35 件）
    連番 技術項目 単一焦点要約 出典
    2021 ALLOCATE 基本構文 ALLOCATE (省略形 ALLOC) は TSO セッションから DD を動的に割り当てる基幹コマンド。JCL の DD 文に相当し、その後の TSO/ISPF/プログラム実行で参照可能になる。 BAS, MF, ZOS31
    2022 DATASET( ) オペランド 割り当てるデータセット名を指定。完全修飾名は単一引用符で囲み、囲まない場合はプレフィックス (PROFILE PREFIX) が付加される。 BAS, ZOS31
    2023 FILE( ) オペランド DD 名 (ddname) を指定する。FILE と DDNAME は同義。プログラムが OPEN する際に参照する名前。省略すると SYSnnnn が自動生成される。 BAS, ZOS31
    2024 NEW オペランド 新規データセットを作成。SPACE/UNIT/DCB 等の割当パラメータが必要。同名データセットが既存ならエラー。 BAS, ZOS31
    2025 OLD オペランド 既存データセットを排他で割り当て。書込み・更新を伴う用途。 BAS, ZOS31
    2026 SHR オペランド 既存データセットを共有で割り当て。複数ジョブ/ユーザが同時に読込可能。読み専用処理の基本。 BAS, ZOS31
    2027 MOD オペランド 既存データセットに追記モードで割り当て。EOF にポジションされ、書込みすると既存末尾に追加。データセットが存在しなければ NEW として扱われる。 BAS, ZOS31
    2028 CATALOG オペランド (DISP 終了処理) 割当解除時にデータセットをカタログ登録するよう指示。JCL DISP=(...,CATLG) に相当。NEW で割り当てた場合の典型指定。 BAS, ZOS31
    2029 DELETE オペランド (DISP 終了処理) 割当解除時にデータセットを削除する。JCL DISP=(...,DELETE) に相当。一時利用後の自動掃除に使う。 BAS, ZOS31
    2030 KEEP オペランド (DISP 終了処理) 割当解除時にデータセットを保持 (非カタログ) する。VOLUME 直接アクセスを想定する場合に使う。 BAS, ZOS31
    2031 PASS オペランド (DISP 終了処理) 後続ステップへ受け渡し用に保持。TSO では同一セッション内で再 ALLOCATE 時に PASS された情報を参照できる。 BAS, ZOS31
    2032 UNCATALOG オペランド 割当解除時にカタログ登録を解除する。データセットの実体は KEEP と同様に残る。 BAS, ZOS31
    2033 SPACE( pri, sec ) オペランド 1 次・2 次の領域量を指定。単位は TRACKS / CYLINDERS / BLOCK( ) 等のキーワードで決まる。例: SPACE(10,5) TRACKS。 BAS, ZOS31
    2034 TRACKS オペランド SPACE の単位をトラックに指定。小さなデータセットでよく使う。 BAS, ZOS31
    2035 CYLINDERS オペランド (CYL) SPACE の単位をシリンダーに指定。中〜大規模データセット向け。 BAS, ZOS31
    2036 BLOCK( ) オペランド SPACE の単位を平均ブロック長 (バイト) で指定。装置依存を避け、論理単位で容量を表現できる。 BAS, ZOS31
    2037 DIR( ) オペランド PDS のディレクトリ・ブロック数を指定。PDS 作成時のみ必須。1 ブロックで約 5〜21 メンバ (メンバ名長による)。 BAS, ZOS31
    2038 RECFM( ) オペランド レコード形式 (F/FB/V/VB/U/FBA など) を指定。プログラムの OPEN 時の整合性が必要。 BAS, ZOS31
    2039 LRECL( ) オペランド 論理レコード長 (バイト) を指定。RECFM=V/VB の場合は 4 バイトの RDW を含めた長さ。 BAS, ZOS31
    2040 BLKSIZE( ) オペランド 物理ブロック長を指定。0 を指定するとシステムが装置最適値を計算 (System Determined Blocksize)。 BAS, ZOS31
    2041 UNIT( ) オペランド 装置タイプ・装置グループ・特定アドレスを指定。例: UNIT(SYSDA), UNIT(3390), UNIT(VIO)。 BAS, ZOS31
    2042 VOLUME( )/VOL( ) オペランド 特定の VOLSER を指定。カタログを介さず直接ボリュームを指定するときに使う。 BAS, ZOS31
    2043 RETPD( ) オペランド 保持期間 (日数) を指定。指定日数経過まで削除されない。EXPDT との同時指定不可。 BAS, ZOS31
    2044 EXPDT( ) オペランド 満了日 (YYYY/DDD または YYYYDDD) を指定。指定日まで削除されない。 BAS, ZOS31
    2045 USING( ) オペランド ATTRIB コマンドで定義済みの属性リストを参照。DCB を 1 ヶ所で定義して再利用するパターン。 BAS, AS1
    2046 LIKE( ) オペランド 既存データセットの DCB 属性をテンプレートとして継承。SPACE は LIKE で継承されない (別途指定要)。 BAS, AS1
    2047 MODEL DSCB GDG ベース等、モデル DSCB を参照することで同一 DCB を引き継がせる古典手法。LIKE が代替可能。 AS1, ZOS31
    2048 REUSE オペランド 同名 DD が既に割り当てられている場合に自動的に FREE してから再割当。エラー回避の常套句。 BAS, ZOS31
    2049 DSORG( ) オペランド データセット組織 (PS/PO/DA など) を指定。通常は RECFM 等から自動決定されるが明示指定で安全側へ。 BAS, ZOS31
    2050 DSNTYPE(LIBRARY) オペランド PDSE を明示作成する指定。DIR は不要 (動的拡張)。SMS 環境では PDSE がデフォルトの場合あり。 AS1, ZOS31
    2051 DSNTYPE(EXTREQ/EXTPREF) 拡張属性付きデータセット (Extended Format) を要求/優先で作成。大容量・ストライプ向け。 AS1, ZOS31
    2052 STORCLAS/MGMTCLAS/DATACLAS SMS クラス指定。SMS 管理データセットでは ACS ルーチンの代わりに直接指定できる。 AS1, ZOS31
    2053 SYSOUT( ) オペランド SYSOUT 装置への動的割当。例: SYSOUT(A)。プログラムからの印刷出力用 DD として使う。 BAS, ZOS31
    2054 DUMMY オペランド ダミー DD として割当。読出しは EOF 即時、書込みは破棄。テスト・分岐回避に有用。 BAS, ZOS31
    2055 DUMMY 用 PATH( ) USS パス用の同等指定。USS ファイルを TSO/プログラムに割り当てるときに使う。 AS1, ZOS31
