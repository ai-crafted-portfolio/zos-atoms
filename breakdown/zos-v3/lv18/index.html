    Sysplex / XCF / GRS / CF¶

    製品: z/OS v3 ／ 本ページ件数: 171 件 ／ 出典区分: Redbook ingest 反映済 v3

    Sysplex種別 （6 件）
    連番 技術項目 単一焦点要約 出典
    4911 MONOPLEX 単一 z/OS イメージを 1 つのスプレックスとして扱う最小構成。XCF はシグナリング相手を持たず、CDS は Sysplex CDS のみ使用される。CFRM/SFM 等の付加 CDS は不要で、PLEXCFG=MONOPLEX を COUPLExx に指定する。 MF, BAS
    4912 XCF-Local Mode XCF 機能を起動するが他システムとシグナリングしない単一イメージ構成。MONOPLEX に近いがアプリケーション(例 RRS, GRS Star は不可) 視点で XCF サービスを利用できる点が異なる。PLEXCFG=XCFLOCAL。 MF, BAS
    4913 Base Sysplex 複数 z/OS イメージを XCF シグナリングで結合するが Coupling Facility を持たない構成。シグナリングは CTC または CDS を経由する。GRS Ring は使えるが GRS Star は不可。 MF, BAS, AS1
    4914 Parallel Sysplex Coupling Facility と Sysplex Timer/STP を用いて複数 z/OS イメージをデータシェアリング可能な形で結合した構成。CF Structure を介して GRS Star, RACF, DB2 Data Sharing, IMS Shared Queue 等を実現する。 MF, BAS, AS1, AS2
    4915 PLEXCFG オプション一覧 COUPLExx PARMLIB の PLEXCFG パラメータは MONOPLEX / XCFLOCAL / MULTISYSTEM / ANY を指定可能。MULTISYSTEM は Base/Parallel 双方を含み、ANY は IPL 時の他システムの存在で動的決定する非推奨指定。 BAS, ZOS31
    4916 Sysplex 名 (SYSPLEX キーワード) COUPLExx の SYSPLEX(name) で最大 8 文字の Sysplex 名を定義する。Sysplex CDS に格納され、複数 LPAR が同じ Sysplex 名を持つことで結合される。一度活性化した Sysplex 名は CDS 再作成までは変更不可。 BAS, ZOS31
    COUPLExx （11 件）
    連番 技術項目 単一焦点要約 出典
    4917 COUPLExx の役割 IEASYSxx の COUPLE= 指定で参照される PARMLIB メンバ。Sysplex 名、PLEXCFG、各種 CDS 配置、CLASSDEF/PATHIN/PATHOUT/LOCALMSG、CFRM/SFM 等の起動有無を一括定義する。NIP フェーズで読み込まれる。 BAS, ZOS31
    4918 DATA TYPE(SYSPLEX) ステートメント Sysplex CDS のプライマリ/オルタネートを定義する。例: DATA TYPE(SYSPLEX) PCOUPLE(SYS1.PCOUPLE) ACOUPLE(SYS1.ACOUPLE)。最低限必要な CDS 指定。 BAS, ZOS31
    4919 DATA TYPE(CFRM/SFM/ARM/WLM/LOGR/BPXMCDS) 各機能用 CDS を同じ DATA TYPE() 構文で個別に登録する。指定された機能のみ NIP/IPL 後に有効化される。CDS が無い機能はそのサービスが非活性となる。 BAS, ZOS31
    4920 PATHIN / PATHOUT 指定 XCF シグナリングの入力/出力経路を CTC (DEVICE) または CF 構造 (STRNAME) で指定する。例: PATHOUT STRNAME(IXC_DEFAULT_1)、PATHIN DEVICE(0E20,0E21)。複数記述可。 BAS, AS1
    4921 CLASSDEF (Transport Class) XCF Transport Class を定義する。CLASSDEF CLASS(DEFAULT) CLASSLEN(956) MAXMSG(2000) GROUP(UNDESIG) のように、長さ・最大メッセージ数・対応グループを指定する。 BAS, AS1
    4922 LOCALMSG 同一 LPAR 内 XCF クライアントへのメッセージング用ローカルバッファ。LOCALMSG MAXMSG(n) で予約量を指定する。性能チューニング項目。 BAS, AS1
    4923 INTERVAL / OPNOTIFY XCF の Status Monitor インターバル(秒)とオペレータ通知タイムアウト。例: INTERVAL(85) OPNOTIFY(90)。Sysplex Failure Management 未使用時のシステム障害検出時間に影響。 BAS, ZOS31
    4924 CLEANUP(n) システム除去時の XCF メンバ後処理タイムアウト(秒)。デフォルト 15。長すぎると Sysplex Wait 状態が長引く。 BAS, ZOS31
    4925 MAXMSG (System Default) PATHOUT/CLASSDEF 個別指定が無い場合の既定 MAXMSG 値。Sysplex 規模に応じて増量するチューニング対象。 BAS, AS1
    4926 RETRY シグナリング経路エラー時のリトライ回数。RETRY(10) が一般的デフォルト。経路品質に応じ調整。 BAS
    4927 COUPLExx の動的反映 SETXCF COUPLE,PCOUPLE=/ACOUPLE= や SETXCF MODIFY によって一部要素は IPL 不要で反映可能だが、PLEXCFG・Sysplex 名・INTERVAL の一部は IPL マイグレーションが必要。 BAS, ZOS31
    Sysplex Time （6 件）
    連番 技術項目 単一焦点要約 出典
    4928 Sysplex Timer (9037) の位置づけ 外部時刻源として複数 LPAR の TOD クロックを同期させる旧世代ハードウェア。Parallel Sysplex 必須要件として ETR ネットワークを構成する。Z14 以降では STP に置き換えが進む。 MF, BAS
    4929 STP (Server Time Protocol) の概要 9037 を不要とするメッセージベースのサーバ間時刻同期。CPC 間 ICA SR/Coupling リンクで Timing-Only/Timing-and-Coupling として構成し、Coordinated Timing Network (CTN) を形成する。 MF, BAS, AS2
    4930 CTN ID と Timing Network STP 構成内で共通の CTN ID(英数 8 文字)を持つ CPC が CTN を形成する。複数 CTN を混在させると同期しないため、Sysplex メンバ全 CPC で同一 CTN ID が必須。 MF, BAS
    4931 Stratum レベル (S0/S1/S2) S0=タイムソース PTS/BTS、S1=直接同期、S2=多段同期。Preferred Time Server(PTS), Backup Time Server(BTS), Arbiter を割り当て、PTS 障害時に BTS が引き継ぐ。 MF, BAS
    4932 Leap Second / Time Offset STP は閏秒の事前/事後挿入や UTC オフセット指定を HMC からスケジュール可能。Sysplex 全体で同一時刻を保つため、システム別の TOD 調整は禁止。 MF, BAS
    4933 TOD クロック同期失敗の影響 Sysplex メンバ間で TOD 差が許容を越えると、XCF ステータス更新の時系列整合が崩れ、Wait State X'0A2' または Sysplex Partition が発生する。 MF, BAS, AS2
    Couple Data Set （14 件）
    連番 技術項目 単一焦点要約 出典
    4934 CDS とは Sysplex 全体で共有する制御情報を格納する VSAM 形式の DASD データセット。プライマリと(できれば物理的に別ボリュームの)オルタネートを必ずペアで構成する。 MF, BAS, ZOS31
    4935 Sysplex CDS システム参加状況、XCF グループ/メンバ、ハートビート情報を保持する最重要 CDS。Sysplex CDS が両系喪失すると Sysplex は停止する。 BAS, ZOS31
    4936 CFRM CDS CFRM ポリシー (CF 定義/STRUCTURE 定義) を保持。CFRM CDS 不在時は CF を利用するサービス (GRS Star, DB2 GBP 等) が起動できない。 BAS, AS1
    4937 SFM CDS Sysplex Failure Management ポリシー (SYSTEM/RECONFIG/PROMPT/ISOLATETIME/CONNFAIL/MEMSTALLTIME 等) を格納する。SFM 未活性時は IXC402D の手動応答に依存する。 BAS, AS1
    4938 ARM CDS Automatic Restart Management ポリシーを格納する。ARM が定義した要素(STC/JOB)を障害時に同一/別システムで自動再起動する。 BAS, AS1
    4939 WLM CDS WLM サービス定義 (Service Policy / Classification Rules / Service Class) を格納する。Sysplex 全 LPAR で同一 WLM ポリシーを共有する。 BAS, AS1
    4940 LOGR CDS System Logger ログストリーム定義および接続/オフロード状態を保持する。LOGR CDS は CFRM とほぼ常時必須で、SYSLOG/OPERLOG/CICS LSR/RRS の前提となる。 BAS, AS1
    4941 BPXMCDS (OMVS) CDS z/OS UNIX Shared File System (Shared HFS/zFS) の所有権・マウント情報を Sysplex 内で共有するための CDS。BPXPRMxx の SYSPLEX(YES) と組で使用する。 BAS, ZOS31
    4942 Primary / Alternate / Spare 全 CDS は PRIMARY と ALTERNATE をペアで用意し、運用中の SPARE CDS を SETXCF で切替可能とする 3 面構成が推奨される。 BAS, ZOS31
    4943 プライマリ喪失時の動作 プライマリ CDS I/O エラー時は自動的にオルタネートをプライマリへ昇格させ、オルタネートが空席となる。SPARE があれば SETXCF COUPLE,ACOUPLE= で補充する。 BAS, ZOS31
    4944 CDS 配置のベストプラクティス プライマリ/オルタネートを別 DASD サブシステム・別チャネルパスに配置し、Sysplex CDS と他 CDS をできるだけ別ボリュームに分散して I/O 集中と単一障害点を排除する。 BAS, AS1
    4945 DISPLAY XCF,COUPLE 全 CDS の現用名・スペア・I/O 状況を表示する。例: D XCF,COUPLE,TYPE=(SYSPLEX,CFRM,SFM)。日常運用での標準確認コマンド。 BAS, ZOS31
    4946 FORMAT TYPE 一致原則 プライマリ/オルタネートはバージョン・容量(ITEM)・FORMAT パラメータが完全一致している必要がある。不一致だと SETXCF COUPLE,ACOUPLE が拒否される。 BAS, ZOS31
    4947 CDS の暗号化対応 z/OS Data Set Encryption により CDS も暗号化可能。鍵を扱う ICSF が IPL 早期に活性化される必要があり、IPL 順序とキャッシュ鍵セットの要件に注意する。 BAS, ZOS31
    CDS サイズ計算 （5 件）
    連番 技術項目 単一焦点要約 出典
    4948 IXCL1DSU の役割 Couple Data Set フォーマット用バッチユーティリティ。DATA TYPE と ITEM パラメータでサイズを宣言し、SHAREOPTIONS(3,3) の VSAM LDS として作成する。 BAS, ZOS31
    4949 Sysplex CDS の ITEM MAXSYSTEM (最大システム数)、MAXGROUP (最大 XCF グループ数)、MAXMEMBER (グループあたり最大メンバ数) の 3 つを IXCL1DSU の DATA TYPE(SYSPLEX) で宣言する。 BAS, ZOS31
    4950 CFRM CDS の ITEM POLICY, CF, STR, CONNECT(POLICY/CF/STR/CONN/SMREBLD/MSGBASED) の各上限を ITEM NAME 指定で宣言する。STR 数を増やすと将来の構造追加に余裕ができる。 BAS, AS1
    4951 SFM / ARM / LOGR の ITEM SFM=POLICY/SYSTEM、ARM=POLICY/RESTART_GROUP/ELEMENT、LOGR=LSR(LogStream)/DSEXTENT/SMDUPLEX 等を ITEM で指定する。下回ると追加定義が拒否される。 BAS, AS1
    4952 再フォーマットの必要性 ITEM 上限を増やすには新しい CDS を IXCL1DSU で作成し、SETXCF COUPLE,ACOUPLE で組込→SETXCF COUPLE,PSWITCH でプライマリ昇格させる無停止切替が標準手順。 BAS, ZOS31
    CFRM （6 件）
    連番 技術項目 単一焦点要約 出典
    4953 CFRM の役割 Coupling Facility リソース(CF と Structure)の管理ポリシーを Sysplex 全体に配布する仕組み。IXCMIAPU で管理データを CFRM CDS に登録し、SETXCF START,POLICY で活性化する。 BAS, AS1
    4954 IXCMIAPU ユーティリティ CFRM/SFM/ARM/LOGR ポリシー定義用バッチユーティリティ。DATA TYPE(CFRM) を指定し、DEFINE POLICY NAME(...) と DEFINE CF, DEFINE STRUCTURE 等のステートメントを記述する。 BAS, AS1, ZOS31
    4955 CFRM ポリシーの活性化 SETXCF START,POLICY,TYPE=CFRM,POLNAME=xxx で活性化。複数ポリシーを CDS に保持して切替えできる。STOP では既存接続は維持される。 BAS, AS1
    4956 CF DEFINE (NAME/TYPE/PARTITION/CPCID/SIDE) Coupling Facility 自体の定義。NAME(CF 論理名)、TYPE(CF モデル)、PARTITION(LPAR ID)、CPCID(CPC シリアル)、SIDE(物理位置) で個々の CF イメージを識別する。 BAS, AS1
    4957 MSGBASED Protocol CFRM 制御を CDS ベースから MSGBASED(XCF メッセージベース) に切替えるとポリシー操作の応答が速くなる。z/OS 1.8 以降の推奨。SETXCF MODIFY,CFRM,MSGBASED=YES。 BAS, AS1, ZOS31
    4958 DUPLEX 制御 Structure 単位で DUPLEX(ENABLED/ALLOWED/DISABLED) を指定し、System-Managed Duplexing の対象を制御する。Lock/List 構造は MSGBASED Duplexing が推奨される。 BAS, AS1
    CFRM Structure （10 件）
    連番 技術項目 単一焦点要約 出典
    4959 STRUCTURE NAME 16 文字以内の構造論理名。製品が要求する名称規則(例 ISGLOCK, IRLMLOCK1, DSNDB0G_GBP1) に合わせて命名する。命名は連結アプリ側のリンクキーとなる。 BAS, AS1
    4960 SIZE / INITSIZE INITSIZE で初期割当、SIZE で最大割当を指定する。CFSizer ツールで算出した値を反映するのが基本で、INITSIZE<SIZE の場合は ALLOWAUTOALT で自動拡張可能。 BAS, AS1
    4961 MINSIZE ALLOWAUTOALT 環境で AutoAlter が縮小可能な下限を指定。指定しない場合は INITSIZE がデフォルト下限となる。 BAS, AS1
    4962 PREFLIST (Preference List) 構造を割り当てる CF の優先順序リスト。先頭から順に試行され、容量・接続性を満たす最初の CF が選ばれる。Rebuild 時は先頭優先で再配置される。 BAS, AS1
    4963 EXCLLIST (Exclusion List) 同じ CF 上に置きたくない構造の共存禁止リスト。可用性向上のため Lock/List のペアを別 CF に分散する目的で使用する。 BAS, AS1
    4964 REBUILDPERCENT Connectivity Failure 時に Rebuild を実行する判定閾値(%)。生き残ったコネクションの比率がこの値以上なら Rebuild、未満なら接続切断とする。 BAS, AS1
    4965 ALLOWAUTOALT 実行時に構造サイズや Entry/Element 比を WLM/XES が自動調整することを許可する。CFSizer 推奨値からの逸脱を吸収するチューニング省力化機能。 BAS, AS1
    4966 DUPLEX(ENABLED/ALLOWED/DISABLED) System-Managed Duplexing の許可レベル。ENABLED は自動二重化、ALLOWED はオペレータ要求で二重化、DISABLED は二重化禁止。 BAS, AS1
    4967 ENFORCEORDER PREFLIST の先頭 CF にあくまで割り付ける制御。可用性より配置安定を優先したい場合に使う。AutoAlter/Rebuild との挙動差異に注意。 BAS, AS1
    4968 CFLEVEL 要件 構造種別ごとに最低 CFLEVEL が要求される(例 List Notification Vector 拡張は CFLEVEL>=15)。PREFLIST 上の全 CF が要件を満たすことが必要。 BAS, AS1, ZOS31
