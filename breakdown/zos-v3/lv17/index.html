    プログラミング言語 (COBOL / PL/I / LE)¶

    製品: z/OS v3 ／ 本ページ件数: 220 件 ／ 出典区分: Redbook ingest 反映済 v3

    COBOL/4 DIVISION 構造 （4 件）
    連番 技術項目 単一焦点要約 出典
    4691 IDENTIFICATION DIVISION プログラム識別情報を記述する必須 DIVISION。PROGRAM-ID のみ必須で、AUTHOR・INSTALLATION・DATE-WRITTEN・DATE-COMPILED・SECURITY・REMARKS はオプション (新規格では多くがコメント扱い)。 MF, BAS
    4692 ENVIRONMENT DIVISION 実行環境と外部装置との接続を記述する DIVISION。CONFIGURATION SECTION (機種定義) と INPUT-OUTPUT SECTION (ファイル定義) から成る。 MF, BAS
    4693 DATA DIVISION プログラムが扱う全てのデータを宣言する DIVISION。FILE SECTION / WORKING-STORAGE SECTION / LOCAL-STORAGE SECTION / LINKAGE SECTION / REPORT SECTION から成る。 MF, BAS
    4694 PROCEDURE DIVISION 実際の処理ロジックを記述する DIVISION。SECTION・段落 (paragraph)・文 (sentence) の階層構造を持ち、USING 句でサブプログラム引数を受け取る。 MF, BAS
    COBOL/IDENTIFICATION DIVISION 段落 （6 件）
    連番 技術項目 単一焦点要約 出典
    4695 PROGRAM-ID プログラム名を定義する唯一の必須段落。リンケージエディタへ渡される外部名となり、CALL の対象名となる。COMMON / INITIAL / RECURSIVE 属性を指定可能。 MF, BAS
    4696 AUTHOR 作成者名を記述するコメント段落。Enterprise COBOL ではコンパイラに認識されるがコメント扱い。社内ヘッダ規約として残されることが多い。 MF, BAS
    4697 INSTALLATION 組織名・部署名を記述するコメント段落。歴史的経緯で残るが現行コンパイラはコメントとして処理する。 MF, BAS
    4698 DATE-WRITTEN プログラム作成日を記述するコメント段落。フォーマットは自由文字列。バージョン管理システムが普及した現代では空または定型句のみ。 MF, BAS
    4699 SECURITY プログラムの機密区分・取扱い注意事項を記述するコメント段落。例: SECURITY. NON-CONFIDENTIAL. MF, BAS
    4700 REMARKS プログラムの説明・備考を自由文で記述する旧来の段落。Enterprise COBOL では非推奨だが過去の資産で多用される。 MF, BAS
    COBOL/CONFIGURATION SECTION （3 件）
    連番 技術項目 単一焦点要約 出典
    4701 SOURCE-COMPUTER コンパイルを行う計算機を宣言する段落。IBM-370 等を記述。WITH DEBUGGING MODE を付けると D 行 (デバッグ行) が有効化される。 MF, BAS
    4702 OBJECT-COMPUTER 実行対象機を宣言する段落。MEMORY SIZE / SEGMENT-LIMIT / PROGRAM COLLATING SEQUENCE を指定し、コード照合順序を決定する。 MF, BAS
    4703 SPECIAL-NAMES 実装定義名 (機能名) とユーザ名の対応・通貨記号・小数点コンマ化・コンソール名・記号文字を宣言する段落。例: DECIMAL-POINT IS COMMA / CURRENCY SIGN IS '¥'。 MF, BAS
    COBOL/INPUT-OUTPUT SECTION （2 件）
    連番 技術項目 単一焦点要約 出典
    4704 FILE-CONTROL プログラムが扱う全てのファイルを SELECT 文で宣言する段落。論理ファイル名と物理 DDname・ファイル編成・アクセスモードを対応付ける。 MF, BAS
    4705 I-O-CONTROL RERUN チェックポイント・SAME RECORD AREA・SAME SORT AREA・MULTIPLE FILE TAPE 等の入出力最適化指定を行う段落。現代では使用頻度が低い。 MF, BAS
    COBOL/SELECT 句 （12 件）
    連番 技術項目 単一焦点要約 出典
    4706 ASSIGN 論理ファイル名を物理装置・DDname に割当てる句。z/OS では ASSIGN TO ddname の形式が標準。装置クラス UT/DA/IS 接頭辞は無視され、最後の8文字以内が DDname となる。 MF, BAS
    4707 ORGANIZATION SEQUENTIAL 順編成ファイル。QSAM (Queued Sequential Access Method) を使用しレコードが書込み順に並ぶ。最も基本的なファイル編成。 MF, BAS
    4708 ORGANIZATION INDEXED 索引付き編成 (VSAM KSDS)。RECORD KEY 句で主キーを指定し、ALTERNATE RECORD KEY で副キーを定義する。 MF, BAS
    4709 ORGANIZATION RELATIVE 相対編成ファイル (VSAM RRDS)。レコードを 1 から始まるレコード番号で直接アクセスする。RELATIVE KEY 句が必要。 MF, BAS
    4710 ACCESS MODE SEQUENTIAL 先頭から順番にアクセスする標準モード。順編成は常にこのモード、索引・相対編成でも順アクセス可能。 MF, BAS
    4711 ACCESS MODE RANDOM キー指定によるランダムアクセス。索引編成・相対編成のみ指定可能。READ で常に直接アクセスする。 MF, BAS
    4712 ACCESS MODE DYNAMIC 順アクセスとランダムアクセスを混在させるモード。READ NEXT で順アクセス、READ で直接アクセスを切替えできる。 MF, BAS
    4713 RECORD KEY 索引編成 (KSDS) の主キーを指定する句。レコード内のキー項目を 01 レベルで参照し、ファイル定義の DATA DIVISION 記述と整合させる。 MF, BAS
    4714 ALTERNATE RECORD KEY 索引編成の副キーを定義する句。WITH DUPLICATES で重複許可。VSAM の AIX (代替索引) と PATH に対応する。 MF, BAS
    4715 RELATIVE KEY 相対編成のレコード番号を保持する整数項目を指定する句。WORKING-STORAGE または LINKAGE の項目を参照する。 MF, BAS
    4716 FILE STATUS I/O 操作後の状態コードを格納する 2 桁文字項目を指定する句。'00'=正常、'10'=EOF、'23'=該当レコードなし、'35'=ファイルなし、'93'=オープン失敗 等。VSAM 用に拡張ステータスを返す 6 文字版もある。 MF, BAS
    4717 PASSWORD VSAM のレベル別パスワード (READ/UPDATE/CONTROL/MASTER) を指定する句。現在は RACF による保護が標準のため使用されない。 MF
    COBOL/DATA DIVISION の SECTION （4 件）
    連番 技術項目 単一焦点要約 出典
    4718 FILE SECTION ファイルレコードの構造を FD/SD 句で宣言する SECTION。OPEN/CLOSE 時のみ領域が割当てられる。 MF, BAS
    4719 WORKING-STORAGE SECTION プログラム実行中に静的に確保される作業領域を宣言する SECTION。複数回呼出されても値を保持し続ける (非 INITIAL/RECURSIVE 時)。 MF, BAS
    4720 LOCAL-STORAGE SECTION 呼出しごとに動的に確保され、呼出し終了時に解放される自動変数領域。RECURSIVE プログラムや CICS でのリエントラント実行で使用する。 MF, BAS
    4721 LINKAGE SECTION 呼出し元から渡されるパラメータを宣言する SECTION。PROCEDURE DIVISION USING 句で受け取り、自身では領域を確保しない (ポインタ受渡し)。 MF, BAS
    COBOL/レベル番号 （5 件）
    連番 技術項目 単一焦点要約 出典
    4722 01 レベル レコード全体を表す最上位レベル。FD/SD 配下またはセクション直下に置く。01 レベル間で REDEFINES が可能。 MF, BAS
    4723 02-49 レベル (基本/グループ項目) 01 配下のサブグループや基本項目を表すレベル。番号は厳密な順序ではなく階層関係のみを表す。慣習的に 05/10/15/20 と 5 刻みにする。 MF, BAS
    4724 66 RENAMES 既存項目群に別名を与える特殊レベル。01 レベル直下に置き THRU で範囲を指定。バイト位置のオーバーレイ名として使う。 MF
    4725 77 独立項目 WORKING-STORAGE/LOCAL-STORAGE/LINKAGE SECTION でグループに属さない基本項目専用のレベル。FILE SECTION では使用不可。 MF, BAS
    4726 88 条件名 親項目の特定値に対する条件名を定義する特殊レベル。SET 文で値を代入したり IF 文で真偽判定したりできる。VALUE 句必須。 MF, BAS
    COBOL/ファイル記述 （3 件）
    連番 技術項目 単一焦点要約 出典
    4727 FD (File Description) FILE SECTION 直下でファイルレコードの物理的属性を記述する句。BLOCK CONTAINS / RECORD CONTAINS / LABEL RECORDS / DATA RECORDS / RECORDING MODE 等を含む。 MF, BAS
    4728 SD (Sort Description) SORT/MERGE 文で使用する内部ソートファイル用の記述。FD と似るが BLOCK CONTAINS と LABEL RECORDS を持たない。 MF, BAS
    4729 RECORD CONTAINS レコード長を指定する句。固定長は単一値、可変長は最小〜最大の範囲を指定。DEPENDING ON 句で実行時可変も可能。 MF, BAS
    COBOL/PIC 句 （13 件）
    連番 技術項目 単一焦点要約 出典
    4730 PIC X 英数字 1 バイトを表す文字。X(20) は 20 バイトの文字列。USAGE DISPLAY と組合せ EBCDIC として格納される。 MF, BAS
    4731 PIC 9 数字 1 桁を表す文字。9(5) は 5 桁の数字。USAGE DISPLAY なら 1 桁=1 バイト (EBCDIC F0-F9 のゾーン)。 MF, BAS
    4732 PIC S 符号付き数値を示す PIC 文字。S は文字数にカウントされない。SIGN IS LEADING/TRAILING SEPARATE 句で符号位置と独立化を制御。 MF, BAS
    4733 PIC V 暗黙の小数点を示す PIC 文字。文字数にカウントされず物理領域は取らない。例: PIC 9(3)V99 は整数3桁・小数2桁。 MF, BAS
    4734 PIC A 英字専用 1 文字を示す。実用例はほとんどなく X で代用される。 MF
    4735 PIC B 編集用空白挿入文字。例: PIC X(2)BX(2) は 2 文字+空白+2 文字。 MF
    4736 PIC 0 編集用ゼロ挿入文字。例: PIC 99099 は中央に固定の 0 を挿入。 MF
    4737 PIC P 暗黙のゼロを表すスケーリング文字。例: PPP9 は実値の 1/1000、9PP は実値の 100 倍を表現する。 MF
    4738 PIC Z ゼロ抑制文字。先頭の 0 を空白に置換する編集句。例: PIC ZZ9 は ' 9' '012' のような表示。 MF, BAS
    4739 PIC $ 通貨記号挿入。先頭固定または浮動。SPECIAL-NAMES の CURRENCY SIGN で別記号 (¥ 等) に変更可能。 MF, BAS
    4740 PIC + / PIC - 符号編集文字。固定位置または浮動配置可能。+ は正負両方を、- は負のみ符号を表示する。 MF, BAS
    4741 PIC CR / PIC DB Credit / Debit 末尾編集文字。値が負のとき末尾に CR または DB を付加する。会計伝票での借方・貸方表記用。 MF
    4742 PIC * ゼロのアスタリスク置換 (チェックプロテクション)。例: PIC ,9.99 は小切手金額の前ゼロを * で置換。 MF
    COBOL/USAGE 句 （14 件）
    連番 技術項目 単一焦点要約 出典
    4743 USAGE DISPLAY ゾーン形式 (EBCDIC 文字) で格納するデフォルト USAGE。数値は 1 桁 1 バイト。最終バイトの上位 4 ビットが符号 (C+/D-/F無符号)。 MF, BAS
    4744 USAGE COMPUTATIONAL (COMP) 2/4/8 バイトの二進整数形式。PIC 9(4) なら 2 バイト、9(9) なら 4 バイト、9(18) なら 8 バイト。z/OS では COMP = COMP-4 = BINARY。 MF, BAS
    4745 USAGE COMP-1 32 ビット単精度浮動小数点数。z/OS では旧来 IBM 16 進浮動 (HFP) だが、コンパイラオプション FLOAT(NATIVE|S390|AFP) で 2 進 IEEE への切替えが可能。 MF
    4746 USAGE COMP-2 64 ビット倍精度浮動小数点数。財務計算ではなく科学技術計算用途。 MF
    4747 USAGE COMP-3 (PACKED-DECIMAL) パック 10 進形式。1 バイトに 2 桁、最終ニブルに符号 (C+/D-/F無符号)。z/OS のハードウェア命令 (AP/SP/MP/DP) が直接演算する高速形式。 MF, BAS
    4748 USAGE COMP-4 COMP と同義。2/4/8 バイト二進整数。範囲は PIC の桁数によって制限される (PIC 9(4) なら 0〜9999)。 MF
    4749 USAGE COMP-5 ネイティブ 2 進整数。COMP-4 と異なり PIC の桁数制限を受けず、領域のフル範囲 (符号付き int16/32/64) を使う。C との連携で重要。 MF, ZOS31
    4750 USAGE BINARY COMP-4 の別名。2 進整数。 MF
    4751 USAGE INDEX OCCURS 配列の添字保持用の 4 バイト二進整数。SET 文でのみ更新可能で算術演算が制限される。 MF
    4752 USAGE POINTER 4 バイト (AMODE 24/31) または 8 バイト (AMODE 64) のアドレスを保持。SET addr-of / SET ptr TO ADDRESS OF item で操作。 MF, ZOS31
    4753 USAGE PROCEDURE-POINTER プログラム入口点アドレスを保持。SET pp TO ENTRY 'name' で動的にエントリを取得し CALL pp で起動。 MF
    4754 USAGE FUNCTION-POINTER C 関数を含む関数入口アドレスを保持する Enterprise COBOL 4 以降の USAGE。XPLINK 関数の呼出しに対応。 MF, ZOS31
    4755 USAGE NATIONAL UTF-16 2 バイト/文字の Unicode 形式。PIC N(n) と組合せ多言語処理に使用。 MF, ZOS31
    4756 USAGE OBJECT REFERENCE オブジェクト指向 COBOL でのオブジェクト参照型。クラス指定子付きで型安全に扱える。 MF
    COBOL/データ定義句 （6 件）
    連番 技術項目 単一焦点要約 出典
    4757 88 条件名 親項目が特定値のとき真となる条件名を定義する。例: 05 STATUS PIC X. 88 ACTIVE VALUE 'A'. → IF ACTIVE / SET ACTIVE TO TRUE。 MF, BAS
