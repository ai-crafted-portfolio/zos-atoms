    JCL PROC / 制御¶

    PROC 文 構文 （8 件）
    連番 技術項目 単一焦点要約 出典
    567 PROC 文の役割 プロシジャの開始を宣言。シンボリック・パラメータのデフォルト値を定義可 MF, BAS
    568 PROC 文の書式 //pname PROC sym1=val1,sym2=val2 形式で 1 行目に記述 MF, BAS
    569 pname (プロシジャ名) 1-8 文字。先頭は英字または国別文字 ($,#,@) MF, BAS
    570 カタログ PROC の pname PDS メンバ名 = プロシジャ名となるので一致必須 MF, BAS
    571 PROC 文の位置 (インストリーム) JOB 文の後・最初の EXEC 文の前に置く MF, BAS
    572 PROC 文の位置 (カタログ) PROCLIB メンバの 1 行目。省略するとデフォルト値定義不可 MF, BAS
    573 PROC 文 オペランド省略 シンボリック デフォルトが不要なら // pname PROC のみで可 MF
    574 PROC 文 継続行 シンボリック リストはカンマで改行し継続行で列挙可 MF
    PEND 文 （5 件）
    連番 技術項目 単一焦点要約 出典
    575 PEND 文の役割 インストリーム プロシジャの終了を示す MF, BAS
    576 PEND 文の書式 //[label] PEND の形。オペランドなし MF, BAS
    577 PEND 文の位置 PROC 文と対になり、呼出 EXEC より前に置く MF, BAS
    578 PEND 省略時のエラー インストリーム PROC で PEND を欠くと JCL エラー MF, BAS
    579 カタログ PROC と PEND カタログ式プロシジャでは PEND 文不要 (記述するとエラー) MF, BAS
