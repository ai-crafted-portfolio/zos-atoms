    DFSMS / IDCAMS / VSAM¶

    製品: z/OS v3 ／ 本ページ件数: 250 件 ／ 出典区分: Redbook ingest 反映済 v3

    DEFINE_CLUSTER （39 件）
    連番 技術項目 単一焦点要約 出典
    1771 DEFINE CLUSTER 基本構文 VSAM クラスター (DATA/INDEX の論理単位) を ICF カタログに登録する IDCAMS コマンド。CLUSTER, DATA, INDEX の 3 レベルで属性を指定する。 BAS, AS1, ZOS31
    1772 NAME(entryname) クラスターのエントリ名 (最大 44 文字)。DATA/INDEX レベルにも個別に NAME 指定可能。省略時はシステムが派生名を生成する。 BAS, AS1, ZOS31
    1773 OWNER(ownerid) クラスターの所有者を 8 文字以内で記録する。アクセス制御ではなく LISTCAT 出力の管理用フィールド。 BAS, ZOS31
    1774 INDEXED (KSDS) Key-Sequenced Data Set を指定。キー順インデックス + データの構成で、ランダム/順次アクセス両対応。VSAM 既定。 BAS, AS1, ZOS31
    1775 NONINDEXED (ESDS) Entry-Sequenced Data Set。書き込み順 (RBA) に格納される。レコードの追加は末尾のみ、論理削除なし。 BAS, AS1, ZOS31
    1776 NUMBERED (RRDS) Relative Record Data Set。レコード番号 (1〜) で直接アクセスする固定長レコード形式。 BAS, AS1, ZOS31
    1777 LINEAR (LDS) Linear Data Set。レコード境界を持たない 4KB CI のバイト列。Data-in-Virtual (DIV) や DB2 表領域で使用。 BAS, AS1, ZOS31
    1778 REUSE / NOREUSE REUSE はクラスターを再利用可能にする。OPEN OUTPUT/RESET で内容を初期化して再使用できる。バッチ作業用領域に好適。 BAS, AS1, ZOS31
    1779 SPEED 初期ロード時の制御区間 (CI) を事前フォーマットしない。高速ロードだが途中で異常終了すると再ロードが必要。 BAS, AS1, ZOS31
    1780 RECOVERY (既定) 初期ロード前に CI を事前フォーマット。中断しても途中までの結果は保全されるが、ロード時間は長い。 BAS, AS1, ZOS31
    1781 RECORDSIZE(avg max) 平均レコード長と最大レコード長 (バイト) を指定。可変長なら avg<max、固定長なら avg=max。SPANNED 時は CI を超えうる。 BAS, AS1, ZOS31
    1782 KEYS(length offset) KSDS / AIX のキー長と、レコード先頭からのオフセットを指定。length+offset は RECORDSIZE 最小値以下である必要。 BAS, AS1, ZOS31
    1783 CONTROLINTERVALSIZE / CISZ(n) 制御区間サイズ (バイト)。512 の倍数 (≤8KB) または 2KB の倍数 (≤32KB)。LDS は 4096 固定。省略時はシステム最適化。 BAS, AS1, ZOS31
    1784 CISZ の選択指針 ランダム多用なら小さめ (バッファ多めに乗る)、順次多用なら大きめ (I/O 回数削減)。一般に 4KB〜18KB が現実的。 BAS, AS1
    1785 BUFFERSPACE(n) 最小バッファ領域 (バイト)。DATA + INDEX の I/O バッファ最低量。指定なしは CISZ ベースの既定。 BAS, AS1, ZOS31
    1786 FREESPACE(ci% ca%) CI 内および CA 内に確保する空き率 (%)。挿入を見込む KSDS では 10〜20% 程度が典型。両方 0 は静的データ向け。 BAS, AS1, ZOS31
    1787 SHAREOPTIONS(cr cs) クロスリージョン共有 (cr) とクロスシステム共有 (cs) のレベル。(1,3)=読取多重/書込排他、(2,3)=読書混在、(3,3)=全責任ユーザ、(4,4)=フル共有。 BAS, AS1, ZOS31
    1788 SHAREOPTIONS(3,3) の注意 整合性は完全にユーザ責任。VSAM はバッファ無効化を保証しない。RLS や DB2 経由なしで複数更新を許す唯一の運用は危険。 BAS, AS1
    1789 SHAREOPTIONS(4,4) と RLS (4,4) は読み取りごとにバッファをリフレッシュ。RLS (Record Level Sharing) は CF 上のロック管理で本格的な共有更新を実現する。 BAS, AS1
    1790 IMBED (廃止) シーケンスセットインデックスをデータ CA の先頭に埋め込む旧オプション。z/OS 1.3 以降は無視される。 BAS, ZOS31
    1791 REPLICATE (廃止) インデックスレコードを 1 トラックに複製配置していた旧オプション。現行 z/OS では効果なし。 BAS, ZOS31
    1792 UNIQUE / SUBALLOCATION (廃止) SMS 以前の VSAM データスペース管理方式。SMS 管理ボリュームでは指定不可、非 SMS でも非推奨。 BAS, ZOS31
    1793 EXCEPTIONEXIT(modname) I/O エラー時に制御を受け取るユーザ出口モジュール名。AMP=AMORG 系の障害処理に使うが現代では稀。 BAS, ZOS31
    1794 CYLINDERS(p s) 1 次/2 次割り振りをシリンダー単位で指定。大量データの典型単位。最大 123 拡張まで取得可能。 BAS, AS1, ZOS31
    1795 TRACKS(p s) 1 次/2 次割り振りをトラック単位で指定。小規模クラスター向け。 BAS, AS1, ZOS31
    1796 RECORDS(p s) 見込みレコード数で割り振り。VSAM が RECORDSIZE と CISZ から必要シリンダー数を計算する。サイズ感に自信がない時に有用。 BAS, AS1, ZOS31
    1797 MODEL(entryname) 既存クラスターの属性を雛形として複製。差異だけを上書き指定するとパラメータ記述を大幅に短縮できる。 BAS, AS1, ZOS31
    1798 VOLUMES(volser ...) 割り振り対象ボリュームのリスト。SMS 管理時は不要 (STORCLAS / ACS が決定)。非 SMS では必須。 BAS, AS1, ZOS31
    1799 UNIT(unittype) 装置型 (3390/SYSDA 等) の指定。SMS 管理時は STORCLAS が選択するため通常不要。 BAS, ZOS31
    1800 MGMTCLAS(name) SMS 管理クラスを明示指定。マイグレーション/バックアップ/世代管理ポリシーを上書き設定する。 BAS, AS1, ZOS31
    1801 STORCLAS(name) SMS ストレージクラスを明示指定。性能/可用性/配置要件を表現し、ACS と組み合わせて配置先プールを決定する。 BAS, AS1, ZOS31
    1802 DATACLAS(name) SMS データクラスを明示指定。RECFM/LRECL/SPACE/VSAM 属性のテンプレート。明示属性指定はデータクラスを上書きする。 BAS, AS1, ZOS31
    1803 SPANNED レコードが複数 CI にまたがって格納可能。RECORDSIZE が CISZ より大きいケースを許容する。インデックス CI は対象外。 BAS, AS1, ZOS31
    1804 TO(yyyyddd) / FOR(days) 保持期限の指定。TO は絶対日付 (ユリウス)、FOR は登録日からの日数。期限内の削除は PURGE 指定が必要。 BAS, AS1, ZOS31
    1805 CATALOG(catname) 登録先カタログを明示指定。省略時は STEPCAT/JOBCAT/マスターまたは ICF カタログ環境の標準解決順。 BAS, ZOS31
    1806 ATTEMPTS(n) パスワード入力試行回数 (0〜7)。RACF 普及前のパスワード保護機構。現代環境では非推奨。 BAS, ZOS31
    1807 AUTHORIZATION(modname [str]) ユーザ認可ルーチン名と任意の文字列パラメータ。旧式の独自認可機構で SAF/RACF に置換済み。 BAS, ZOS31
    1808 CODE(code) パスワード入力プロンプトに表示するコード文字列 (1〜8 文字)。旧式パスワード保護の構成要素。 BAS, ZOS31
    1809 CONTROLPW / MASTERPW / UPDATEPW / READPW 操作レベル別の旧式パスワード (制御/マスター/更新/読取)。RACF データセットプロファイルに完全置換され、現行では使用しない。 BAS, ZOS31

    1818 DEFINE GDG 基本 Generation Data Group のベースエントリを ICF カタログに作成。世代データセットの命名・管理単位となる。 BAS, AS1, ZOS31
    1819 LIMIT(n) GDG が保持する世代数の上限 (1〜255、EXTENDED 時 999)。超過すると最古世代が NOEMPTY 時は単独、EMPTY 時は全てロールオフ。 BAS, AS1, ZOS31
    1820 SCRATCH / NOSCRATCH ロールオフ時に世代を物理削除 (SCRATCH) か、カタログから外すのみ (NOSCRATCH) か。SCRATCH が一般的。 BAS, AS1, ZOS31
    1821 EMPTY / NOEMPTY LIMIT 到達時に全世代をロールオフ (EMPTY) するか、最古 1 件のみ (NOEMPTY、既定) するか。 BAS, AS1, ZOS31
    1822 EXTENDED GDG 世代数上限を 255 から 999 に拡張する z/OS V1R12 以降の属性。既存 GDG を ALTER で EXTENDED 化することも可能。 BAS, ZOS31
    1823 OWNER(id) GDG ベースの所有者識別。LISTCAT 上の管理用情報。 BAS, ZOS31
    1824 TO / FOR (GDG) GDG ベース自体の保持期限。世代側ではなくベースの期限。期限切れ後は通常の DELETE が可能。 BAS, ZOS31
    1825 PURGE / NOPURGE DELETE 時に期限内世代も削除するか。PURGE は期限を無視、NOPURGE は期限内エラーとする。 BAS, ZOS31
