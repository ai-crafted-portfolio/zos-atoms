    #4446 DVIPA の役割

    検証手順

    検証目的: 「DVIPA の役割」(DVIPA) の構文・パラメータ・運用上の役割を z/OS Communications Server マニュアルと実機で確認し、VTAM / TCP/IP オペレータが正しく定義・コマンド操作できることを検証する。

    [セッション環境]
TSO ISPF EDIT — PROFILE.TCPIP の VIPADYNAMIC ブロック編集

[コマンド 1] PROFILE.TCPIP の VIPADYNAMIC ブロックを ISPF EDIT で開く
  EDIT 'TCPIP.PROFILE(TCPIP)'    (VIPADYNAMIC ブロック)

[オプション説明]
  DVIPA            - Dynamic VIPA (動的 Virtual IP Address)
  VIPADEFINE       - 主 DVIPA 定義
  VIPABACKUP       - バックアップ DVIPA 定義
  VIPADISTRIBUTE   - Sysplex Distributor 配信先
  対象             - DVIPA の役割
  (出典: z/OS Communications Server IP Configuration Reference (DVIPA章)、原文ママ)

[出力 1]
  VIPADYNAMIC
    VIPADEFINE [MOVEABLE|...] 255.255.255.0 [DVIPA アドレス]
    VIPABACKUP [...] [DVIPA アドレス]
    VIPADISTRIBUTE [DVIPA] PORT [...] DESTIP [...]
    [DVIPA の役割 に関連するステートメント]
  ENDVIPADYNAMIC

[コマンド 2] V TCPIP,,OBEYFILE で動的反映 + D TCPIP,,SYSPLEX,VIPADYN で確認
  V TCPIP,TCPIP,OBEYFILE,USER1.TCPIP(DVIPADEF)
  D TCPIP,TCPIP,SYSPLEX,VIPADYN

[出力 2]
  EZZ0053I COMMAND VARY OBEYFILE COMPLETED SUCCESSFULLY
  EZZ8260I SYSPLEX CS V[...] [DVIPA] [状態]
  (★ 反映確認 — DVIPA が ACTIVE / MOVING / BACKUP 等の正しい状態で表示されれば成功)

[出典抜粋]
  SIOCSVIPA/SIOCSVIPA6 ioctl or BIND explicitly to DVIPA. If within VIPARANGE, DVIPA created dynamically. POrt=portnum (1-65535) — Applications bound to DVIPA and this port number on target stacks are paused from sysplex distributor workload balancing.
  (出典: z/OS Communications Server マニュアル、原文ママ)

    前提条件: TCP/IP スタック (TCPIP プロシージャ) 稼働中、PROFILE.TCPIP 参照権限、TCP/IP オペレータ ID 権限、関連デーモン (PAGENT/syslogd/RESOLVER 等) 稼働

    プレースホルダ: [対象項目]=DVIPA の役割、[TCPIP プロシージャ名]=サイト指定 (既定 TCPIP)、[VTAMLST メンバ名]=サイト指定、[リソース名]=サイトで実在する Major Node / APPL / PU / LU 等 (具体値は実機・サイト標準に従う)

    記録項目:

    • ① 出典マニュアル (z/OS Communications Server IP Configuration Reference) の「DVIPA の役割」該当章のページ番号と抜粋
    • ② 実機で投入したコマンド全文 (パラメータ/メンバ名/オペランド値含む)
    • ③ EZZ/EZY/EZD メッセージ ID と本文 (z/OS console / SDSF.LOG 採取)
    • ④ 実行前 / 実行後 の対象リソース状態 (D NET / D TCPIP / NETSTAT の差分)
    • ⑤ 異常時のエラーメッセージ全文と RC (出た場合)
    • ⑥ 実機環境情報 (z/OS / Comm Server リリース / TCPIP・VTAM プロシージャ名 / オペレータ ID / 実行時刻)

    理解度チェック

    問題: 「DVIPA の役割」(DVIPA) の説明として正しいものはどれか?

    • A. Dynamic VIPA、物理インタフェースに依存しない論理 IP、Sysplex 移動可能
    • B. VIPADYNAMIC/ENDVIPADYNAMIC で DVIPA 関連を囲む
    • C. DVIPA を本拠地スタックに定義 (MOVEABLE/NONDISRUPTIVE 等)
    • D. 他スタックでバックアップ DVIPA を定義、Rank 順に引継ぎ

    正解: A

    解説: 正解 (A) は「DVIPA の役割」の単一焦点要約 (出典: z/OS Communications Server マニュアル、v3 抽出時 一次資料 由来)。他選択肢は同じ Lv2「DVIPA」内の隣接項目要約であり、「DVIPA の役割」の説明としては誤り。実機 D NET / D TCPIP / NETSTAT で検証し、出典章で詳細確認推奨。
