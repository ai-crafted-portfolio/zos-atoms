    #4046 Logstream 利用の前提

    検証手順

    検証目的: SMF Logstream における「Logstream 利用の前提」の位置付けと典型操作 (summary: MVS Logger (IXGLOGR) が稼働、CFRM または DASD-only ログ) を実機で確認する。

    [セッション環境]
MVS Console, SYS1.PARMLIB, System Logger (LOGR CDS)

[コマンド 1] (PARMLIB) SMFPRMxx を LOGSTREAM 記録に変更
[オプション説明]
  RECORDING(LOGSTREAM)         - 記録先
  DEFAULTLSNAME(IFASMF.DEFAULT)
  LSNAME(IFASMF.PERF,TYPE(30,89))

[出力 1] (D LOGGER,L,LSN=IFASMF.PERF)
  IXG601I LOGSTREAM ATTRIBUTES + STATUS 表示

[コマンド 2] SET SMF=xx (新 PARMLIB 適用)
[出力 2]
  IFA040I SMF NOW USING SMFPRM[XX]
  IFA842I DUPLICATE LOGSTREAM ... REJECTED (重複定義時)
  (出典: z/OS MVS Initialization and Tuning Reference SMFPRMxx LSNAME, 原文ママ)

    前提条件: System Logger 稼働、LOGR CDS 定義済、IFASMF.* ログストリーム DEFINE 済、SAF FACILITY 権限

    プレースホルダ: [XX]=PARMLIB suffix 2 文字

    記録項目:

    • ① LOGR CDS 定義 (IXCMIAPU 出力)
    • ② SMFPRMxx の DEFAULTLSNAME / LSNAME 設定
    • ③ D LOGGER,L,LSN=IFASMF.* の状態
    • ④ IFASMFDL での log stream ダンプ結果

    理解度チェック

    問題: SMF Logstream における「Logstream 利用の前提」の説明として最も適切なものはどれか?

    • A. MVS Logger (IXGLOGR) が稼働、CFRM または DASD-only ログ
    • B. LOGR ポリシーで Logstream を DEFINE
    • C. オフロード閾値 (高水位)
    • D. オフロード後の目標水位

    正解: A

    解説: (A) は 一次資料 「Logstream 利用の前提」の本来の説明: MVS Logger (IXGLOGR) が稼働、CFRM または DASD-only ログ と一致。 (B)(C)(D) は同じ Lv2 「SMF Logstream」内の隣接項目 (Logstream 定義 IXCMIAPU 等) の説明であり対象機能が異なる。
