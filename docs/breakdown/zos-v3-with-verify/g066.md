    **検証手順**

    検証目的: 「Activity trace configuration file, mqat.ini」 (構成 (QM / Queue / Channel)) の IBM MQ 9.3 仕様・動作を出典マニュアル (MQ 9.3 構成ガイド) + 実機 runmqsc 等で確認し、MQ 管理者・開発者が正しく利用・運用できることを検証する。


    前提条件: IBM MQ 9.3 検証環境構築済 (QMGR1 = 動作中のキュー マネージャ)、mqm グループ 所属 ユーザ で ログイン、LDAP/TLS/AMS は 任意機能 として 別途構成

    プレースホルダ: [QMGR1]=検証用 キュー マネージャ 名 (任意命名可)、[NNNN]=AMQxxxx メッセージ 中の 環境依存 番号、[host1]/[host2]=multi-instance ノード ホスト 名 (該当時)

    記録項目:

    - (1) 対象 Lv3 「Activity trace configuration file, mqat.ini」 の マニュアル (MQ 9.3 構成ガイド) 該当 ページ (188) 引用ブロック の キャプチャ
    - (2) runmqsc QMGR1 セッション の 入出力 (DISPLAY/ALTER の AMQxxxxI メッセージ 含む)
    - (3) Lv2 「構成 (QM / Queue / Channel)」 で 期待される 副作用 (errors ログ / FDC / accounting キュー 等) 確認
    - (4) 出典: 公式資料 (188) と summary page (188) の 整合性 検証
    - (5) 異常時 動作 (権限不足・パラメタ 誤り 等) の AMQxxxxE / MQRC_xxxx ハンドリング 確認

    ---

    **理解度チェック**

    問題: IBM MQ 9.3 (構成 (QM / Queue / Channel)) における 「Activity trace configuration file, mqat.ini」 の説明として 出典マニュアル (MQ 9.3 構成ガイド p.188) と 整合する のは どれ か?

    - A. Activity trace configuration file, mqat.ini — Activity trace configuration file, mqat.ini: The activity trace configuration file, mqat.ini, is used to configure activ…
    - B. IBM MQ configuration file, mqs.ini に関する 説明
    - C. Security exits in MQIPT に関する 説明
    - D. Console message monitoring に関する 説明

    正解: **A**

    解説: 出典マニュアル (MQ 9.3 構成ガイド p.188) によれば、「Activity trace configuration file, mqat.ini」 は (A) の通り。 (B)(C)(D) は同 Lv2 「構成 (QM / Queue / Channel)」 内の 別 Lv3 項目で、本問の 「Activity trace configuration file, mqat.ini」 とは 異なる。 (出典: MQ 9.3 構成ガイド p.188、原文ママ)
